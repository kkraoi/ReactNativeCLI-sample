# 社内配布について
- GooglePlayストアに公開しない
- Android端末に直接APKを作ってインストールする
- 野良アプリ/インハウス配布
- 公式：https://reactnative.dev/docs/signed-apk-android
- AABではなく、APKを作る

## 1. アップロードキー（キーストア）の生成
- `.keystore`：アプリが本物であることを証明する証明鍵
    - GooglePlayストアはじつはこの中身を審査することはない
    - インターネット上の「暗号化通信の証明書（SSL証明書など）」を作るための仕組みを流用しているため、世界基準で厳格に「お前はどこのどいつだ」を登録する
- パスワード/エイリアスは必ずメモする
    - この手の管理方法は？
        - `android/local.properties`にパスワードなどの情報を記載する。
        - `android/local.properties`：PCごとのローカル環境依存のパス（Android SDKの場所など）を書き、gitには絶対に入れない、という目的のファイル
- windowsの場合：keytoolを実行して生成する（コマンド実行）
    - プロジェクト直下にkeystoreができていたら、android\app\testapp.keystoreに持っていく
- -keystoreの名称
    - `[プロジェクト名].keystore`や`[社名]-[アプリ名].keystor`でよい
- -aliasの名称
    - `testapp-app-key`
    - local.propertiesのRELEASE_KEY_ALIASと同じ
- keystoreの有効期限
    - `-validity`で指定。
    - 期限が切れると、
        - ビルド（正確には署名）ができなくなる
        - 更新インストールができなくなる
        - OSが拒否する
    - 10000：10000日 => 27年
    - 期限の確認コマンド
        - `keytool -list -v -keystore [あなたのキーストア名].keystore`

```powershell
keytool -genkeypair -v -storetype PKCS12 -keystore プロジェクト名.keystore -alias プロジェクト名-app-key -keyalg RSA -keysize 2048 -validity 10000
```
↑プロンプトと遭遇する
- キーストアのパスワードを入力してください
    - local.propertiesに書いたパスワード
- 新規パスワードを再入力してください
    - 上記と同じパスワード
    - 確認用です。2つとも同じものを入力します。
- 姓名は何ですか
    - testapp-developer または 社名や開発チーム名
    - 個人名ではなく、アプリ名やチーム名を入れるのが一般的です
- 組織単位名は何ですか
    - Development または 空欄でEnter
    - 部署名（Dev、DXなど）を入れますが、空欄でもOK。
- 組織名は何ですか
    - あなたの会社名やチーム名
    - 会社名などを入れますが、個人開発ならアプリ名でOK
- 都市名または地域名は何ですか
    - Minato-kuまたは Tokyo など
    - 市区町村名です
- 都道府県名または州名は何ですか
    - Tokyo
    - 都道府県名です
- この単位に該当する2文字の国コードは何ですか
    - JP
    - ここだけは必ず「JP」と大文字2文字で入れてください
- 〜〜 でよろしいですか
    - はい または yes または y
    - 確認です。これを入れると鍵が生成されます。


```local.properties
RELEASE_STORE_PASSWORD=your_secure_password
RELEASE_KEY_PASSWORD=your_secure_password
RELEASE_KEY_ALIAS=my-key-alias

# RELEASE_STORE_PASSWORD と RELEASE_KEY_PASSWORD は同じで良い
# RELEASE_STORE_PASSWORD：.keystore という「鍵が入っている保管庫（ファイル）」を開くためのパスワード
# RELEASE_KEY_PASSWORD：保管庫の中にある「特定の署名鍵（エイリアス）」を実際に使ってアプリに署名するためのパスワード
```

## 2. 署名設定の設定
- アプリをビルドするとき、自動で.keystoreを使って署名されるよう、設定ファイルを書き換える。
- `android/app/build.gradle`を編集する
    1. ファイルの最上部に「読み込み処理」を追加する
        - パスワードやエイリアスを読み込む
    2. 署名設定を書き換える
        - signingConfigs をファイル内検索
    3. リリース署名を有効にする
        - buildTypes をファイル内検索

```.gradle
// 1. ファイルの最上部に「読み込み処理」を追加する
// android/app/build.gradle の最上部（1行目）に追加
if (localPropertiesFile.exists()) {
    localPropertiesFile.withReader('UTF-8') { reader ->
        localProperties.load(reader)
    }
    
    // 🔥 [オプション] ファイルはあるが、必要なキーが空っぽの場合もガードする
    if (localProperties.getProperty('RELEASE_STORE_PASSWORD') == null) {
        throw new GradleException(
            "🛑 [Error] 'local.properties' は存在しますが、リリース用の設定（RELEASE_STORE_PASSWORDなど）が足りません。\n" +
            "local.properties.example を参考に、必要なパスワード群を追記してください。"
        )
    }
} else {
    // 🔥 ファイル自体が存在しない場合は、ビルドを強制終了してメッセージを出す
    throw new GradleException(
        "🛑 [Error] 'local.properties' ファイルが見つかりません。\n" +
        "プロジェクトのルートディレクトリに 'local.properties' を作成し、必要なリリース署名情報を設定してください。\n" +
        "詳細はチームのドキュメント、または 'local.properties.example' を確認してください。"
    )
}

// 2. 署名設定を書き換える
signingConfigs {
    debug {
        // （既存のdebug設定はそのまま触らなくてOK）
    }
    
    // 👇 ここを書き換え・追記
    release {
        if (localProperties.getProperty('RELEASE_STORE_PASSWORD') != null) {
            // 先ほど android/app/ 配下に置いたキーストアファイルの名前を指定します
            storeFile file("YOUR_PROJECT_NAME.keystore") 
            storePassword localProperties.getProperty('RELEASE_STORE_PASSWORD')
            keyAlias localProperties.getProperty('RELEASE_KEY_ALIAS')
            keyPassword localProperties.getProperty('RELEASE_KEY_PASSWORD')
        }
    }

// 3. リリース署名を有効にする
buildTypes {
        debug {
            signingConfig signingConfigs.debug
        }
        release {
            // 既存の設定（minifyEnabled など）がある場合はそのまま残す
            
            // 👇 この1行があること（または追加）を確認
            signingConfig signingConfigs.release

            // signingConfig signingConfigs.debugになっていたら👆に書き直す
        }
    }
```

## 3. ビルドコマンド実行
- package.jsonにscriptを登録
- 画面の最後に `BUILD SUCCESSFUL` と表示されれば成功
- `プロジェクト/android/app/build/outputs/apk/release/app-relase.apk`ができる

```json
"scripts": {
    ...
    
    // 👇 これを追記します（apkとaabの違いを明記）
    "build:android-apk": "cd android && gradlew assembleRelease",
    "build:android-aab": "cd android && gradlew bundleRelease"

    // 👇 macの場合はこれ
    "build:android-apk": "cd android && ./gradlew assembleRelease",
    "build:android-aab": "cd android && ./gradlew bundleRelease"
  },
```
